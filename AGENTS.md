# AGENTS.md

## 项目概览

这是一个基于 Vue 3 和 Vite 的单页前端项目，项目名为 `orbit-love-page`。

当前项目的目标非常明确：生成一个中文恋爱主题落地页，视觉上偏精致、柔和、私密，内容围绕“两个人的故事”展开。页面不是通用内容平台，而是一个高度定制化的展示页模板，适合替换文案后直接作为纪念页、告白页或情侣主页使用。

从当前代码看，这个项目具备以下特征：

- 单页面应用，没有路由
- 没有后端
- 没有状态管理库
- 没有测试框架
- 没有 TypeScript
- 没有组件拆分，核心逻辑集中在一个 Vue 单文件组件中

## 技术栈

- Vue 3
- Vite 5
- `@vitejs/plugin-vue`
- 原生 CSS

`package.json` 中可见的脚本如下：

- `npm run dev`：启动本地开发服务器
- `npm run build`：构建生产版本
- `npm run preview`：本地预览构建结果

## 目录结构

当前仓库结构比较简单，主要文件如下：

- `package.json`：项目依赖与脚本
- `package-lock.json`：npm 锁文件
- `vite.config.js`：Vite 配置，当前仅启用 Vue 插件
- `index.html`：入口 HTML，定义页面标题并引入 Google Fonts
- `src/main.js`：Vue 应用入口，挂载 `App.vue`
- `src/App.vue`：项目核心文件，包含页面数据、交互逻辑、模板结构和大量样式
- `src/style.css`：仅保留非常基础的全局 reset
- `dist/`：已存在构建产物目录

## 页面结构

页面主要由以下几个区块组成：

1. 顶部导航
2. Hero 主视觉区
3. 故事时间线 `timeline`
4. 专属情书 `letter`
5. 未来清单 `future`
6. 页脚 `footer`

导航锚点直接跳转到页面内部区块，没有使用 Vue Router。

## 核心实现方式

### 1. 内容配置集中在 `profile`

`src/App.vue` 中定义了一个 `profile` 响应式对象，几乎承载了整页所有业务内容。它目前包含：

- 页面品牌标题
- Hero 文案
- 双方名字
- 专属暗号
- 坐标文案
- 开始日期
- 页脚引用语
- 关键词列表
- 时间线列表
- 情书内容
- 承诺列表
- 未来计划列表

这意味着：

- 这个项目当前更像“单文件模板”
- 调整页面文案时，优先修改 `profile`
- 如果后续需要支持多套内容，应该优先把 `profile` 抽离成独立配置文件或 JSON 数据源

### 2. 计算属性负责展示层派生数据

当前使用了多个 `computed` 来生成界面展示值，例如：

- 截取前四个关键词
- 拼接双方名字
- 处理中心文案换行
- 格式化纪念日日期
- 提取情书正文段落
- 生成年份与页脚文本

这说明代码已经有一定“数据与展示分离”的意识，但仍然全部放在同一个组件中。

### 3. 动画与滚动效果较轻量

项目没有引入动画库，而是使用原生方式实现：

- `requestAnimationFrame`：用于“已经一起走过”天数的数字递增动画
- `IntersectionObserver`：用于 `.fade-up` 元素进入视口时淡入上移

这类实现比较轻，适合当前项目体量。

### 4. 样式主要写在 `App.vue`

虽然项目有 `src/style.css`，但它只负责基础 reset。真正的视觉系统都在 `App.vue` 的 `<style>` 中，包括：

- 颜色变量
- 卡片面板样式
- 按钮样式
- Hero 区布局
- 时间线样式
- 情书与未来清单样式
- 淡入动画
- 响应式断点

因此后续维护时应默认认为：

- `App.vue` 是内容中心
- `App.vue` 也是视觉中心

## 视觉设计特征

当前页面风格比较统一，主要特征如下：

- 粉白色系渐变背景
- 柔和玻璃拟态面板
- 中英混排字体
- 大号衬线标题
- 轻量滚动动效
- 明显的纪念感与私人定制感

字体来自 `index.html` 中的 Google Fonts：

- `Instrument Serif`
- `Manrope`
- `Noto Serif SC`

这说明项目当前依赖外部字体 CDN。若部署环境无法访问 Google Fonts，页面字体表现会退化到系统回退字体。

## 当前交互逻辑

项目的交互非常有限，属于静态展示页增强型交互，主要包括：

- 点击导航跳转到对应区块
- Hero 区按钮跳转到情书或未来清单
- 页面初次加载后自动播放“天数增长”动画
- 页面滚动时区块渐入显示

没有表单、没有用户输入、没有接口请求、没有本地存储。

## 日期与文案逻辑

### 日期处理

`App.vue` 中存在两个和日期有关的函数：

- `formatDate(value)`：把日期格式化为 `YYYY.MM.DD`
- `countDaysSince(value)`：按“自然日”计算从开始日期到今天经过了多少天

这里使用的是浏览器本地时间，并且按本地午夜做差。含义上更接近“纪念日天数”，而不是精确到小时分钟的时长。

### 文案特征

现有文案是中文，语气偏克制、温柔、长期关系导向，和视觉风格一致。项目的内容不是占位符级别的“Lorem Ipsum”，而是已经写过一版完整文案，因此继续改动时要注意统一语气，不要混入明显不同风格的文本。

## 适合的修改方式

### 如果只是替换展示内容

优先修改：

- `src/App.vue` 中的 `profile`

### 如果只是微调视觉

优先修改：

- `src/App.vue` 中的 `<style>`

### 如果想让结构更易维护

建议按下面方向逐步拆分：

- 将 `profile` 抽到 `src/data/profile.js`
- 将 Hero、Timeline、Letter、Future、Footer 拆成独立组件
- 将主题变量整理到单独样式文件

## 当前项目的限制

从工程角度看，当前项目有几个明显限制：

- 所有主要逻辑都集中在 `src/App.vue`，单文件较大
- 没有测试
- 没有 ESLint / Prettier 配置
- 没有 TypeScript
- 没有内容配置抽离机制
- 没有部署说明
- 没有 README

这不代表项目不可用，而是说明它目前更适合“快速交付一个高质感静态单页”，不太适合直接扩展成复杂多页面应用。

## 后续维护建议

如果未来继续开发，建议优先按以下顺序优化：

1. 补充 `README.md`，写明启动、构建、部署方式
2. 把 `profile` 拆出组件文件
3. 拆分 `App.vue`
4. 增加 lint 与格式化配置
5. 视复杂度决定是否引入 TypeScript

## 对后续协作者最有价值的结论

- 这是一个小型 Vue 3 + Vite 单页项目
- 核心内容和样式高度集中在 `src/App.vue`
- 目前最重要的数据入口是 `profile`
- 项目主要价值在于视觉表达和文案承载，不在复杂业务逻辑
- 如果只是改“你们的故事”，基本不需要改项目结构
- 如果要长期维护，第一步应该先做内容配置抽离和组件拆分
